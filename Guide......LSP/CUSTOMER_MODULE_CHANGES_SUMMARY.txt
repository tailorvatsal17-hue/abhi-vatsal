CHANGE 1: Enhanced create() method

ADDITIONS:
  • Extract user_id from JWT token (req.userId) - not from request body
  • Validate all required fields present

  • Return 201 status instead of 200

BEFORE:

    Booking.create(booking, (err, data) => {
        res.send(data);  // Always 200
    });

AFTER:

    // ... validate all inputs ...
    Booking.create(booking, (err, data) => {
        res.status(201).send(data);  // 201 Created
    });

SECURITY IMPROVEMENTS:
  • User ID now from trusted JWT source, not request body
  • Users cannot forge other user IDs
  • Validation prevents invalid bookings

---

CHANGE 2: Enhanced getById() method

ADDITIONS:
  • Authorization check: user can only view their own bookings
  • Returns 403 Forbidden if not owner

BEFORE:
    Booking.getById(req.params.id, (err, data) => {
        res.send(data);  // No auth check
    });

AFTER:
    const bookingId = req.params.id;
    const userId = req.userId;
    Booking.getById(bookingId, (err, data) => {
        if (data.user_id !== userId) {
            return res.status(403).send({
                message: "Access denied..."
            });
        }
        res.send(data);
    });

SECURITY IMPROVEMENTS:
  • Users cannot view other users' bookings
  • Prevents data disclosure
  • Clear error message

---

CHANGE 3: Completely refactored cancel() method

MAJOR ENHANCEMENTS:
  1. Authorization check (owner verification)
  2. Status validation:
     - Cannot cancel completed bookings
     - Cannot cancel already-cancelled bookings
  3. Business logic validation:
     - Can only cancel 24+ hours before booking
  4. Detailed error messages

BEFORE:
    Booking.cancel(req.params.id, (err, data) => {
        res.send({ message: "Booking cancelled" });
    });

AFTER:
    // Step 1: Get booking
    // Step 2: Check ownership
    if (booking.user_id !== userId) return 403;
    // Step 3: Check status
    if (booking.status === 'Completed') return 400;
    // Step 4: Check 24-hour rule
    if (hoursUntilBooking < 24) return 400;
    // Step 5: Cancel
    Booking.cancel(bookingId, ...);

IMPACT:
  • Robust cancellation logic
  • Clear error messages
  • Business rules enforced
  • Prevents abuse

IMPACT: Booking system now secure and validated
