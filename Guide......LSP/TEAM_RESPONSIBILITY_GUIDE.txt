## Instructions for GitHub
1. **Create a Main Repository**: One member (usually Saurab or Susan) should create the main project repository.
2. **Use Folders**: Organize the files into the `lsp-sd2` structure as shown above.
3. **Commit Messages**: Use clear messages like "Added DB Schema (Vatsal)" or "Updated Layout Pug (Susan)".
4. **Final Check**: Ensure the `.env` file is NOT uploaded (keep it in `.gitignore`) but provide an `env-sample` file.
